## Project center

- This project is centered on my current BTZCL reacting-flow development work.

### Counterflow branch (current foundation)

- Core goal: keep JiCF-style canonical comparison behavior while using OpenFOAM's built-in reacting chemistry stack.
- Outcome target: quantify where OpenFOAM reacting CFD aligns/diverges from the Cantera 1D reference under matched boundary-condition intent.

### Jet-in-crossflow branch (Summer 2026 expansion)

- Core goal: extend reacting-flow validation workflows to 3D reacting jet-in-crossflow (RJICF) configurations relevant to gas-turbine combustors.
- Outcome target: characterize how flame location and heat release modify shear-layer vortices under gas-turbine-relevant pressure and velocity ratios.

## Context anchor

This project is part of a propulsion and energy research ecosystem centered on combustion science and dynamics at Georgia Tech's [Ben T. Zinn Combustion Lab](https://www.comblab.gatech.edu/).

The RJICF work specifically investigates reacting jets in crossflow, a configuration relevant to Rich-Quench-Lean (RQL) combustors used to reduce NOx emissions in gas turbines. The focus is on how exothermic heat release and flame positioning alter large-scale vortical structures such as shear-layer vortices (SLVs) and the counter-rotating vortex pair (CVP), which govern mixing efficiency and emissions performance.

## Workflow map

### Counterflow validation backbone (established)

- **Reference baseline (JiCF-BC-calc-main):**
  - Cantera 1D counterflow runs tune inlet conditions for target density ratio (S) and momentum flux ratio (J).
  - Canonical JSON outputs store boundary conditions and flame-level metrics.
- **OpenFOAM plumbing checks:**
  - Diffusion-only check against analytical complementary error function solution.
  - Passive counterflow case generation and mixing-field validation.
- **OpenFOAM reacting stage:**
  - 2D reacting counterflow setup with chemistry/thermo/reaction properties configured in OpenFOAM.
- **JiCF-style compare loop:**
  - Preset selection and JSON ingestion from canonical reference when available.
  - OpenFOAM output written back in a JiCF-like schema.
  - Automated side-by-side Cantera vs OpenFOAM comparison report.

### Reacting jet-in-crossflow (RJICF) LES expansion (in progress)

- **Problem configuration:**
  - 3D reacting jet injected into crossflow at gas-turbine-relevant pressures and velocity ratios.
  - Two canonical flame regimes:
    - **R1:** Flame outside the shear layer → SLVs present.
    - **R2:** Flame inside the shear layer → SLVs partially or fully suppressed.
- **Numerical approach:**
  - Large-eddy simulation (LES) in OpenFOAM.
  - Single-step, infinitely fast methane reaction model.
  - Refined mesh near the jet shear layer to resolve vortex formation and breakdown.
- **Structured parameter sweep:**
  - Systematic variation of jet velocity \(v_j\), crossflow velocity v_cf, pressure \(P\), and flame position \(\delta\), with other variables held fixed to isolate physical mechanisms.
- **Validation logic:**
  - Compare vortex behavior across R1 vs R2.
  - Assess suppression mechanisms of SLVs and evolution of the CVP.
  - Evaluate turbulence and reaction-model performance against available experimental datasets.

## Why this matters

- **Counterflow work:** moves from single-tool dependence to cross-solver comparison without losing data consistency, preserves canonical case structure so regression and validation remain trackable, and establishes a reusable comparison grammar between Cantera and OpenFOAM.
- **RJICF work:** extends that validation discipline into fully 3D reacting, turbulent combustion regimes, clarifies how combustion heat release alters vortex dynamics under engine-relevant conditions, and supports improved understanding of mixing and NOx mitigation in RQL combustors.
- **Combined pipeline:** creates a solver-verified reacting-flow path that spans 1D canonical validation → 2D reacting counterflow → 3D LES reacting jet systems.
